# 万一 Edge 退出策略

## 两套独立逻辑

梯度只根据当前 Long/Short Edge 查表，决定普通做多或做空目标仓位。成交账本只根据真实成交记录实际入场和平仓 Edge。两者不能混算。

实际入场 Edge：

```text
E_entry = Σ(q_entry × E_entry_fill) / Σq_entry
```

实际平仓 Edge：

```text
E_close_actual = Σ(q_close × E_close_fill) / Σq_close
```

每笔数量相同时，上式等于实际成交 Edge 相加后除以成交次数。行情曾经到达的极值不能代替实际入场成本。

## 减仓后再加仓的成本

采用交易所通用的移动平均持仓成本：

1. 同方向加仓：用剩余仓位成本和新成交重新做数量加权。
2. 反方向减仓：剩余仓位的平均入场 Edge 不变，只按减仓数量同比例移除成本。
3. 减仓后重新同方向加仓：只使用剩余仓位和新成交计算，已经平掉的数量及其历史入场 Edge 不再参与。
4. 穿过零仓位：到零为止结算旧方向，超过零的部分按该笔实际成交 Edge 建立新方向成本。

例：Short `0.010 BTC @ 0.0448%`，先平 `0.004 BTC` 后，剩余 `0.006 BTC` 的入场均价仍是 `0.0448%`。再开 Short `0.002 BTC @ 0.0400%`：

```text
新入场均价
= (0.006 × 0.0448% + 0.002 × 0.0400%) / 0.008
= 0.0436%
```

已平掉的 `0.004 BTC` 不再进入新持仓均价。

参考：[OKX 平仓不改变剩余持仓均价](https://www.okx.com/en-sg/help/how-to-calculate-the-average-transaction-price-of-the-contract)、[Bybit USDT 合约加权入场价与部分平仓](https://www.bybit.com/en/help-center/article/?id=000001066&language=en)。

## 万一条件

“万一”固定表示 `0.0100%` Edge，不是任意正收益。

Short 持仓使用当前可执行 Long Edge 平仓：

```text
E_long_now >= E_short_entry + 0.0100%
```

Long 持仓使用当前可执行 Short Edge 平仓：

```text
E_short_now <= E_long_entry - 0.0100%
```

每次准备下单时只判断当前可执行反向 Edge。实际成交可能有滑点，无法提前控制；成交后的实际平仓均价只记录到界面和日志，不用于阻止下一笔。

## 最大仓位

万一条件成立后，不能固定清零。程序使用当前持仓同方向 Edge 查询该方向梯度，得到当前档位允许保留的最大仓位。

Short 示例：

```text
Short <= 0.050% -> -0.005 BTC
Short <= 0.045% -> -0.010 BTC
Short <= 0.040% -> -0.015 BTC
Short <= 0.035% -> -0.020 BTC
```

当前仓位 `-0.015`、当前 Short Edge `0.0489%` 时，当前档位目标是 `-0.005`。如果当前 Long Edge 同时达到 Short 实际入场均价 `+0.0100%`，程序逐笔买入，最多退到 `-0.005`，不会固定清零。

Long 完全对称。当前 Long Edge 决定允许保留的最大多仓；当前 Short Edge 是否低于 Long 实际入场均价 `0.0100%`，决定是否允许卖出减仓。

## 信号优先级

1. 普通梯度产生正常加仓或反向目标时，严格执行普通梯度。
2. 普通梯度没有产生调仓信号时，才检查万一 Edge 退出。
3. 万一条件成立且当前仓位超过当前档位上限，向档位目标逐笔减仓。
4. 当前仓位已经处于档位上限以内，不下单。
5. 没有匹配的同方向梯度档位时，该方向允许保留仓位为 `0`。

## 日志

普通梯度触发记录当前两条 Edge、使用方向、命中阈值、梯度目标、当前仓位和剩余调仓量。

万一触发记录轮次、持仓方向、当前两条 Edge、实际入场均价、`0.0100%` 万一值、精确触发线、当前可执行平仓 Edge、当前梯度档位目标、当前仓位、本次方向和最大下单量。

实际平仓成交继续记录数量加权平仓 Edge、成交滑点和订单生命周期，供事后复盘。
