# 通用双市场梯度与 Edge 周期策略

本文定义一套可复用于两个交易场所之间的双腿价差策略。它保留当前系统的两部分核心行为：

1. 普通梯度根据实时 Long/Short Edge，把净仓位逐步推向配置的目标仓位；
2. 持仓周期账本根据两腿实际成交计算入场 Edge、平仓 Edge 和已配对收益，在普通梯度没有动作时提供盈利退出。

本文是策略规范，不绑定某一家交易所。为方便描述，下文把两个交易场所称为市场 A 和市场 B。在当前实现中：

- 市场 A = Variational；
- 市场 B = Lighter；
- 正仓位 = 做多 A、做空 B；
- 负仓位 = 做空 A、做多 B。

> 所有 Edge 数值的单位都是百分比的“百分点”。`0.01%` 等于 1 bp，不等于 1%。

## 1. 策略目标

策略不预测单边价格方向，只交易同一资产在两个市场之间的可执行价差。每次动作同时包含两条腿：

| 方向 | 市场 A | 市场 B | 净仓位符号 |
| --- | --- | --- | --- |
| Long | 买入 | 卖出 | 正数 |
| Short | 卖出 | 买入 | 负数 |

策略维护一个带符号目标净仓位：

- `target_qty > 0`：目标是 Long 组合；
- `target_qty < 0`：目标是 Short 组合；
- `target_qty = 0`：目标是平仓。

梯度中的数量是“目标总仓位”，不是“这一笔下单数量”。实际每笔数量由单次下单上限控制。

## 2. Edge 定义

### 2.1 行情输入

设：

- `A_bid`、`A_ask`：市场 A 的可执行买一和卖一；
- `B_bid`、`B_ask`：市场 B 的可执行价格或指定深度 VWAP；
- `R`：两个市场之间的价格映射比例，通常为 1。

使用对称百分比差，避免简单除以单边价格造成方向不对称：

```text
SymmetricEdge(left, right)
    = 200 × (right - left) / (right + left)
```

### 2.2 Long Edge

Long 方向需要买入 A、卖出 B，因此使用 A 的 ask 和 B 的 bid：

```text
Long Edge
    = 200 × (R × B_bid - A_ask)
      / (R × B_bid + A_ask)
```

Long Edge 越高，买 A / 卖 B 越有利。

### 2.3 Short Edge

Short 方向需要卖出 A、买入 B，因此使用 A 的 bid 和 B 的 ask：

```text
Short Edge
    = 200 × (R × B_ask - A_bid)
      / (R × B_ask + A_bid)
```

Short Edge 越低，卖 A / 买 B 越有利。

### 2.4 实际成交 Edge

信号 Edge 只能决定是否尝试下单。收益账本必须使用两条腿的真实成交均价：

```text
Fill Edge
    = 200 × (B_fill - A_fill)
      / (B_fill + A_fill)
```

同一个公式同时用于 Long 和 Short 成交。方向只影响“这个 Edge 对哪一侧更有利”以及收益的正负计算。

## 3. 普通梯度逻辑

### 3.1 配置形式

每档配置为：

```text
Edge阈值:目标净仓位
```

示例：

```bash
GRADIENT_SINGLE_ORDER_QTY=0.0015

GRADIENT_LONG=0.060:0.015,0.065:0.030,0.070:0.045,0.075:0.060,0.080:0.075,0.085:0.090,0.090:0.100

GRADIENT_SHORT=0.040:-0.015,0.035:-0.030,0.030:-0.045,0.025:-0.060,0.020:-0.075,0.015:-0.090,0.010:-0.100
```

Long 阈值升高时，目标仓位必须严格增大。Short 阈值降低时，目标仓位必须严格减小。阈值或目标仓位重复、只填写半行、单次下单量不为正数，都会使策略保持禁用。

### 3.2 Long 梯度选择

当 `Long Edge >= threshold` 时，该档成立。如果同时命中多档，选择最高的已命中阈值。

例如：

| Long Edge | 命中档位 | 目标仓位 |
| ---: | ---: | ---: |
| 0.058% | 无 | 不变 |
| 0.060% | 0.060% | +0.015 |
| 0.068% | 0.065% | +0.030 |
| 0.077% | 0.075% | +0.060 |
| 0.093% | 0.090% | +0.100 |

Long 梯度只有在目标仓位高于当前仓位时才产生动作，动作固定为买 A / 卖 B。

### 3.3 Short 梯度选择

当 `Short Edge <= threshold` 时，该档成立。如果同时命中多档，选择最低的已命中阈值。

例如：

| Short Edge | 命中档位 | 目标仓位 |
| ---: | ---: | ---: |
| 0.043% | 无 | 不变 |
| 0.040% | 0.040% | -0.015 |
| 0.033% | 0.035% | -0.030 |
| 0.028% | 0.030% | -0.045 |
| 0.008% | 0.010% | -0.100 |

Short 梯度只有在目标仓位低于当前仓位时才产生动作，动作固定为卖 A / 买 B。

### 3.4 实际下单量

命中目标后，不会一次性下完全部差额：

```text
delta_qty = abs(target_qty - current_qty)
order_qty = min(single_order_qty, delta_qty)
```

`order_qty` 还要向下对齐市场 B 的最小 lot。每次两腿完成后重新读取仓位和行情，再决定下一笔。

### 3.5 Long/Short 不是固定的开仓/平仓

Long 和 Short 表示交易方向，不表示订单角色。订单是开仓、加仓、减仓还是反向，取决于当前仓位和目标仓位。

#### 示例 A：从零建立 Long 仓位

```text
当前仓位 = 0
Long Edge = 0.068%
命中目标 = +0.030
单次下单 = 0.0015
```

策略反复执行买 A / 卖 B，每次 0.0015，直到仓位达到 +0.030。

#### 示例 B：Long 信号减少 Short 仓位

```text
当前仓位 = -0.020
Long 目标 = +0.015
```

Long 动作仍然是买 A / 卖 B。最初几笔是在平 Short；穿过 0 后，后续订单开始建立 Long。总差额为 0.035，按单次上限逐笔执行。

#### 示例 C：Short 信号减少 Long 仓位

```text
当前仓位 = +0.030
Short 目标 = -0.015
```

Short 动作是卖 A / 买 B。它先平掉 +0.030 Long，再继续建立 -0.015 Short。

#### 示例 D：当前仓位已经越过目标

```text
当前仓位 = -0.011
Short 档目标 = -0.010
```

因为 `-0.010` 不低于当前的 `-0.011`，Short 梯度不会继续卖出。只有更深一档给出低于 `-0.011` 的目标，例如 `-0.015`，才会继续增加 Short。

### 3.6 双方向冲突

如果同一时刻 Long 梯度要求提高仓位，同时 Short 梯度又要求降低仓位，普通梯度不会任选一边，而是不给出普通信号。

在空仓状态下，这意味着不下单。在已有周期仓位时，系统仍可继续检查后文的盈利退出条件，但退出保护只能减仓，不能借冲突机会反向开仓。

## 4. 入场 Edge 与平仓 Edge 账本

### 4.1 只认真实两腿成交

一个订单只有在市场 A 和市场 B 都取得完整成交均价后，才进入周期账本。

- 网络成交推送：直接使用真实成交均价和真实成交数量；
- 手动恢复：只接受用户明确输入的成交均价和数量，并标记为手动来源；
- 禁止根据仓位均价或仓位变化推导成交价格。

这条约束非常重要。仓位只用于校验账本是否一致，不能代替成交事件。

### 4.2 周期定义

空仓后的第一笔完整两腿成交创建新周期：

- 买 A / 卖 B 创建 Long 周期；
- 卖 A / 买 B 创建 Short 周期。

同方向成交增加周期仓位并更新入场成本；反方向成交与已有仓位配对，形成部分或全部平仓；仓位归零时周期完成并归档。

### 4.3 加权入场 Edge

同方向多笔成交后，入场 Edge 按数量加权：

```text
Entry Edge
    = sum(entry_qty_i × entry_edge_i)
      / sum(entry_qty_i)
```

示例：

```text
卖 A / 买 B 0.001，成交 Edge = 0.040%
卖 A / 买 B 0.003，成交 Edge = 0.050%

Short Entry Edge
    = (0.001 × 0.040% + 0.003 × 0.050%) / 0.004
    = 0.0475%
```

### 4.4 加权平仓 Edge

反方向实际成交后，平仓 Edge 按已平数量加权：

```text
Close Edge
    = sum(close_qty_i × close_edge_i)
      / sum(close_qty_i)
```

示例：

```text
买 A / 卖 B 0.001，成交 Edge = 0.050%
买 A / 卖 B 0.002，成交 Edge = 0.056%

Close Edge
    = (0.001 × 0.050% + 0.002 × 0.056%) / 0.003
    = 0.054%
```

没有任何反方向实际成交时，平仓 Edge 不存在。界面或消息可以显示 0，但不能据此生成收益。

## 5. 基于入场成本的盈利退出

### 5.1 Long 周期

Long 周期的入场方向是买 A / 卖 B，退出方向是卖 A / 买 B，因此使用实时 Short Edge 判断能否退出：

```text
Long Edge收益 = Entry Edge - Close Edge

允许退出条件：
Current executable Short Edge
    <= Entry Edge - minimum_profit
```

### 5.2 Short 周期

Short 周期的入场方向是卖 A / 买 B，退出方向是买 A / 卖 B，因此使用实时 Long Edge 判断能否退出：

```text
Short Edge收益 = Close Edge - Entry Edge

允许退出条件：
Current executable Long Edge
    >= Entry Edge + minimum_profit
```

当前默认：

```text
minimum_profit = 0.01%
```

也就是至少 1 bp 的 Edge 优势。

### 5.3 第一次平仓不需要历史平仓 Edge

退出判断使用的是当前反方向可执行 Edge，而不是必须等到已有一笔平仓成交。

例如 Short 周期：

```text
实际 Entry Edge = 0.0448%
当前可执行 Long Edge = 0.0548%
差值 = 0.0100%
```

即使当前还没有平仓成交，条件已经达到 1 bp，也可以发出第一笔买 A / 卖 B 的退出单。成交后才产生实际 Close Edge。

### 5.4 退出数量仍受普通梯度边界约束

成本线退出不是“无条件全部清仓”。它先根据当前普通梯度计算允许退到哪里：

- 当前为 Short 仓位时，退出目标必须满足 `current_qty < target_qty <= 0`；
- 当前为 Long 仓位时，退出目标必须满足 `0 <= target_qty < current_qty`；
- 没有匹配梯度档时，退出边界默认为 0；
- 保护退出本身不会跨过 0 建立反向仓位。

实际每笔退出量仍为：

```text
min(single_order_qty, abs(exit_target - current_qty))
```

## 6. 普通梯度与 Edge 退出的兼容顺序

每次评估按以下顺序执行：

1. 校验策略配置、真实仓位与账本仓位；
2. 运行普通 Long/Short 梯度；
3. 如果普通梯度给出唯一动作，直接使用普通梯度；
4. 只有普通梯度没有动作时，才检查基于 Entry Edge 的盈利退出；
5. 退出信号仍要经过行情确认、两腿平衡、在途订单和下单前复核。

这个优先级意味着普通梯度始终保留控制权。

| 场景 | 普通梯度 | Edge 退出 | 最终行为 |
| --- | --- | --- | --- |
| 普通梯度要求继续加仓 | 有信号 | 即使可盈利退出 | 按普通梯度加仓 |
| 普通梯度要求减仓或反向 | 有信号 | 不检查 | 按普通梯度目标执行 |
| 普通梯度没有动作，成本线达到 1 bp | 无信号 | 有信号 | 按允许边界减仓 |
| 普通梯度没有动作，成本线未达到 1 bp | 无信号 | 无信号 | 等待 |
| 账本与真实仓位不一致 | 禁止 | 禁止 | 停止策略并人工检查 |

### 6.1 部分平仓后允许普通梯度重新加仓

示例：

```text
Short 初始仓位 = -0.010
实际 Entry Edge = 0.0448%

买 A / 卖 B 平仓 0.004
实际 Close Edge = 0.0548%
剩余仓位 = -0.006
```

此时已经有 0.004 数量产生已配对收益。如果随后 Short 普通梯度再次要求 `-0.008`，普通梯度优先，策略可以卖 A / 买 B 加回 0.002。此前已经配对的收益不会撤销；剩余入场成本与新增成交重新加权。

### 6.2 普通梯度可以跨零，退出保护不可以

```text
当前仓位 = -0.002
Long 普通梯度目标 = +0.005
```

普通梯度可以执行总计 0.007 的买 A / 卖 B，先平 Short，再开 Long。

如果同样的买入动作来自成本线退出保护，最大目标只能到 0，不会自动建立 +0.005 Long。

## 7. 信号确认与执行保护

梯度或 Edge 退出产生候选动作后，不会立即无条件下单。

### 7.1 抗尖刺确认

当前实现使用两档确认：

- Edge 样本极差不超过 0.03 个百分点时：连续约 400ms，并覆盖至少 3 个不同报价；
- 波动较大时：连续约 1s，并覆盖至少 4 个不同报价。

方向、阈值、目标仓位或差额变化会重置确认。

### 7.2 下单前复核

确认完成后重新读取两边价格。如果方向、命中档位或目标仓位发生变化，本次信号取消并重新确认。

### 7.3 风险闸

任一条件不满足都不能创建新订单：

- 两腿真实仓位不平衡；
- 账本仓位与真实仓位持续不一致；
- 上一笔策略订单仍在途；
- Var 报价断线或陈旧；
- Lighter 私有账户、仓位或订单簿未就绪；
- 梯度配置无效；
- 策略处于人工停止状态。

## 8. 收益口径

收益必须区分“Edge 收益”和“计价币毛收益”。

### 8.1 Edge 收益

Long 周期：

```text
Edge PnL = Entry Edge - Close Edge
```

Short 周期：

```text
Edge PnL = Close Edge - Entry Edge
```

只对已经被反方向成交配对的数量计算。只有入场、没有平仓时，不存在已实现收益。

### 8.2 精确计价币毛收益

如果四个真实成交价格都存在，可以直接按每单位捕获价差计算。

Long 周期：

```text
entry_unit_spread = B_entry_sell - A_entry_buy
close_unit_spread = A_close_sell - B_close_buy

exact_quote_pnl
    = paired_qty × (entry_unit_spread + close_unit_spread)
```

Short 周期：

```text
entry_unit_spread = A_entry_sell - B_entry_buy
close_unit_spread = B_close_sell - A_close_buy

exact_quote_pnl
    = paired_qty × (entry_unit_spread + close_unit_spread)
```

这是当前首选口径，因为它直接来自真实成交价格，不需要用百分比反推。

### 8.3 估算计价币收益

当缺少可用于精确价差的原始价格、但存在真实 Edge 和参考价格时，可退化为：

```text
estimated_quote_pnl
    = paired_qty × reference_price × Edge PnL / 100
```

估算值必须明确标记为“约”，不能伪装成精确毛收益。

### 8.4 部分平仓、本轮收益与累计收益

- `本轮已配对数量`：当前周期已经被反方向成交抵消的数量；
- `本轮已配对毛收益`：当前周期已配对部分的真实毛收益；
- `累计已配对数量`：已完成周期数量加当前周期已配对数量；
- `累计已配对毛收益`：已完成周期收益加当前周期已配对收益；
- `最近完成周期`：最近一次仓位从非零回到零的完整周期结果。

只有入场时：

```text
close_qty = 0
本轮已配对收益 = 不显示
```

出现第一笔反向成交后，立即开始累计已配对收益，不需要等到整个周期归零。

### 8.5 收益示例：Long 周期

入场：

```text
买 A 0.001 @ 65,000
卖 B 0.001 @ 65,040
entry_unit_spread = 40
Entry Edge ≈ +0.0615%
```

平仓：

```text
卖 A 0.001 @ 65,100
买 B 0.001 @ 65,070
close_unit_spread = 30
Close Edge ≈ -0.0461%
```

收益：

```text
Long Edge PnL ≈ 0.0615% - (-0.0461%) = +0.1076%
精确毛收益 = 0.001 × (40 + 30) = +0.0700u
```

### 8.6 收益示例：Short 周期

入场：

```text
卖 A 0.001 @ 65,000
买 B 0.001 @ 64,970
entry_unit_spread = 30
Entry Edge ≈ -0.0462%
```

平仓：

```text
买 A 0.001 @ 64,900
卖 B 0.001 @ 64,950
close_unit_spread = 50
Close Edge ≈ +0.0770%
```

收益：

```text
Short Edge PnL ≈ 0.0770% - (-0.0462%) = +0.1232%
精确毛收益 = 0.001 × (30 + 50) = +0.0800u
```

### 8.7 收益示例：部分平仓

```text
Short Entry Qty = 0.010
Short Entry Edge = 0.0448%

第一次买 A / 卖 B 0.004，Close Edge = 0.0548%
已配对数量 = 0.004
Edge PnL = 0.0548% - 0.0448% = +0.0100%
剩余仓位 = -0.006
```

此时立即产生 0.004 数量的本轮已配对收益。剩余 0.006 仍保留原入场成本。后续反方向成交继续更新 Close Edge；同方向新成交只重新加权剩余 Entry Edge，不会抹掉已经配对的收益。

### 8.8 毛收益不等于账户净利润

当前账本记录的是双腿成交产生的毛收益，默认未扣除：

- 两边交易手续费；
- 资金费率；
- 借贷或资金占用成本；
- 提现、跨链或兑换成本；
- 未被成交价格反映的其他费用。

通用策略若要输出净利润，应单独记录每条腿的实际费用：

```text
net_pnl
    = exact_quote_pnl
      - fee_A
      - fee_B
      - funding
      - financing_cost
      - other_costs
```

## 9. 周期状态与持久化

同一个持久化文件同时保存：

- 当前未完成周期的方向、仓位、Entry Edge 和 Close Edge；
- 已完成周期的累计数量和累计毛收益；
- 最近的完成周期摘要；
- 下一个周期编号。

仓位归零后只清空当前周期成本，已完成周期和累计收益继续保留。新成交在同一份状态中创建下一个周期。

程序停止期间发生外部平仓时，系统无法猜测成交价格。重启后应保留旧成本并等待用户录入两腿真实成交均价和数量；不允许通过仓位变化推导价格。

## 10. 通用实现检查清单

迁移到其他交易所或其他资产时，至少确认以下事项：

1. A/B 两边价格已经转换到同一计价单位；
2. Long 使用 A ask 与 B bid，Short 使用 A bid 与 B ask；
3. 梯度数量解释为目标净仓位，而不是单笔数量；
4. Long 选择最高已命中阈值，Short 选择最低已命中阈值；
5. 同一时刻只允许一个明确的普通梯度方向；
6. 普通梯度优先，成本线退出只作为无普通动作时的补充；
7. 成本线退出只减仓到允许边界，不自行跨零；
8. 周期账本只消费两腿完整真实成交；
9. Entry/Close Edge 按成交数量加权；
10. 部分平仓立即产生已配对收益；
11. 精确毛收益优先使用四个真实成交价格；
12. 费用与资金成本独立核算；
13. 仓位、账本或两腿状态不一致时 fail closed；
14. 通知、日志和持久化不得阻塞行情与下单路径。
